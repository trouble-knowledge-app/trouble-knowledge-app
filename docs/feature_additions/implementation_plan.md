# タグ付け・検索機能強化の実装計画

ユーザーからの要望に基づき、「タグ付け」「検索機能強化」の2つの機能を実装します。（※「画像の添付」は今回は見送ります）

## 1. データベースの改修 (app.py)

既に稼働している SQLite データベース (`knowledge.db`) に対し、新しいカラムを追加する処理を実装します。

- **カラムの追加**:
  - `tags` (TEXT) : カンマ区切りで保存されたタグ一覧（例: `強度不足,加工ミス`）
- `app.py` 起動時に、既存テーブルにカラムが存在しない場合は自動で `ALTER TABLE` を実行して追加します。

## 2. タグ付け機能と検索機能の強化

- **タグのUI (フロントエンド)**:
  - 「強度不足」「納期遅延」「加工ミス」といったよく使う【おすすめタグ】をボタンとして用意し、クリッカブルでオンオフ切り替えできるようにします。
  - 任意のキーワードも自由に入力してタグ化できる入力欄（Enterで追加）も設けます。
- **検索の強化 (バックエンド/フロントエンド)**:
  - 表示された記録のタグ（バッジ）をクリックすると、自動的にそのタグ単語で検索が行われるようにします。
  - バックエンドの検索機能にて、これまでの「事象」「対応」などの本文検索に加え、`tags` カラムも `LIKE` 検索の対象に含めるように変更します。
  - エクスポート出力（CSV / JSON / TXT）にも「タグ」の情報を項目として追加します。

---

### 実装手順

1. `app.py` データベース初期化(`init_db`)に `tags` カラム追加処理を書き込む。APIルート (POST, GET, Export) を改修。
2. `index.html` にタグ選択/入力用のUIとスタイルクラス追加。
3. `app.js` にて「おすすめタグ」選択ロジック、記録時のタグデータ追加、表示時のタグ生成（クリックで検索含む）を実装。
4. `static/css/style.css` にタグバッジや選択ボタンスタイルを追加。
