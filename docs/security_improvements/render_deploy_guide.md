# RenderでのデプロイおよびAPI_KEY設定手順

GitHubへのコードプッシュが完了しました。
Renderの自動デプロイが設定されている場合、数分待つと新しいコードが本番環境に反映されます。

今回のアップデートで**APIキー認証**が追加されたため、本番環境で安全に利用するためには、Renderのダッシュボード上で環境変数 `API_KEY` を設定する必要があります。（設定しない場合、デフォルトの `change-me-in-production` が使用されてしまい安全ではありません）

以下の手順に従って環境変数を設定してください。

## 環境変数 (API_KEY) の設定手順

1. **Renderのダッシュボードにログイン**します。
   - [https://dashboard.render.com/](https://dashboard.render.com/)

2. **対象のWeb Service（trouble-knowledge-app 等）を選択**します。

3. 左側のメニューから **「Environment」** をクリックします。

4. **「Add Environment Variable」** をクリックします。（既存のEnvironment Variable一覧の下の方にあります）

5. 以下の内容を入力します：
   - **Key**: `API_KEY` (大文字、アンダースコア)
   - **Value**: あなたが自分で決めた任意の文字列（例: `my-secret-password-1234`, `tk-app-admin-999` など、推測されにくいものを設定してください）

   > [!TIP]
   > このバージョンの `render.yaml` にて `API_KEY` は `sync: false` と設定されているため、コード内の設定ファイルには値が保存されず、ダッシュボードからのみ安全に設定できるようになっています。

6. （もし `SECRET_KEY` がまだ設定されていない場合はついでに設定します）
   - **Key**: `SECRET_KEY`
   - **Value**: 「Generate」ボタンがあればそれを押すか、適当なランダム文字列（例: `sk-5f8a0b9c...`）を入力

7. **「Save Changes」** ボタンをクリックして設定を保存します。

8. 環境変数を変更すると、Renderが**自動的に再デプロイ（Restart）を開始**します。デプロイが完了するまで（「Deploy live」と表示されるまで）数分お待ちください。

## 動作確認

デプロイ完了後、サイトURLにアクセスしてみてください。
画面上に「認証が必要です」のダイアログが表示されれば成功です！

手順5で設定した `API_KEY` の値を入力して「ログイン」し、正常に記録の閲覧や追加ができることを確認してください。
